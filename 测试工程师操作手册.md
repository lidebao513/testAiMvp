# AI测试工程师管理平台操作手册

## 1. 平台概述

AI测试工程师管理平台是一个集成了AI技术的测试管理系统，旨在提高测试效率和质量。平台提供了以下核心功能：

- **测试概览**：实时监控测试执行情况，查看测试通过率和覆盖率
- **测试用例管理**：管理AI生成的测试用例，支持预览和编辑
- **执行监控**：实时监控测试执行过程，查看执行日志和失败截图
- **知识库管理**：管理测试模式、缺陷和最佳实践，支持相似用例推荐

## 2. 环境搭建

### 2.1 前端环境搭建

1. **安装Node.js**：确保安装了Node.js 18.0或更高版本
2. **安装依赖**：
   ```bash
   cd frontend
   npm install
   ```
3. **启动开发服务器**：
   ```bash
   npm run dev
   ```
4. **构建生产版本**：
   ```bash
   npm run build
   ```

### 2.2 后端环境搭建

1. **创建虚拟环境**：
   ```bash
   python -m venv testAiMvp
   ```
2. **激活虚拟环境**：
   - Windows：`testAiMvp\Scripts\activate`
   - Linux/Mac：`source testAiMvp/bin/activate`
3. **安装依赖**：
   ```bash
   # 安装知识库服务依赖
   pip install -r services/knowledge-service/requirements.txt
   
   # 安装其他服务依赖
   pip install -r services/generator-service/requirements.txt
   pip install -r services/executor-service/requirements.txt
   pip install -r services/parser-service/requirements.txt
   pip install -r services/self-healing-service/requirements.txt
   ```
4. **启动服务**：
   ```bash
   # 启动知识库服务
   cd services/knowledge-service
   uvicorn app.main:app --reload
   
   # 启动其他服务（在不同终端）
   cd services/generator-service
   uvicorn app.main:app --reload --port 8001
   
   cd services/executor-service
   uvicorn app.main:app --reload --port 8002
   
   cd services/parser-service
   uvicorn app.main:app --reload --port 8003
   
   cd services/self-healing-service
   uvicorn app.main:app --reload --port 8004
   ```

## 3. 登录和配置

### 3.1 登录系统

1. 打开浏览器，访问前端应用地址（默认：http://localhost:5173）
2. 系统默认不需要登录，直接进入测试概览页面

### 3.2 系统配置

1. **API服务配置**：
   - 前端默认配置：在`frontend/src/api/config.ts`中配置后端服务地址
   - 后端服务配置：在各服务的`.env`文件中配置数据库连接等信息

2. **环境变量配置**：
   - `MONGODB_URI`：MongoDB连接地址
   - `MILVUS_URI`：Milvus向量数据库连接地址
   - `OPENAI_API_KEY`：OpenAI API密钥（用于生成嵌入向量）
   - `LOCAL_EMBEDDING_URL`：本地嵌入模型API地址（可选）

## 4. 知识库配置

### 4.1 访问知识库管理

1. 在左侧导航栏点击"知识库管理"
2. 进入知识库管理页面，包含以下功能：
   - 测试模式可视化
   - 历史缺陷检索
   - 相似用例推荐

### 4.2 添加测试模式

1. **通过API添加**：
   ```bash
   curl -X POST http://localhost:8000/api/knowledge/test-patterns \
     -H "Content-Type: application/json" \
     -d '{
       "name": "边界值测试",
       "description": "测试边界条件和极限值",
       "category": "单元测试"
     }'
   ```

2. **通过后端代码添加**：
   ```python
   from app.core.knowledge_base import KnowledgeBase
   
   kb = KnowledgeBase(
       mongo_uri="mongodb://localhost:27017",
       milvus_uri="localhost:19530"
   )
   
   pattern = {
       "name": "边界值测试",
       "description": "测试边界条件和极限值",
       "category": "单元测试"
   }
   pattern_id = kb.add_test_pattern(pattern)
   ```

### 4.3 添加缺陷

1. **通过API添加**：
   ```bash
   curl -X POST http://localhost:8000/api/knowledge/defects \
     -H "Content-Type: application/json" \
     -d '{
       "title": "登录页面响应慢",
       "description": "用户登录时页面响应时间超过3秒",
       "severity": "高",
       "status": "已修复",
       "createdAt": "2026-03-20"
     }'
   ```

2. **通过后端代码添加**：
   ```python
   defect = {
       "title": "登录页面响应慢",
       "description": "用户登录时页面响应时间超过3秒",
       "severity": "高"
   }
   defect_id = kb.add_defect(defect)
   ```

### 4.4 添加最佳实践

1. **通过API添加**：
   ```bash
   curl -X POST http://localhost:8000/api/knowledge/best-practices \
     -H "Content-Type: application/json" \
     -d '{
       "title": "测试命名规范",
       "description": "测试方法名应清晰表达测试目的",
       "category": "测试规范"
     }'
   ```

2. **通过后端代码添加**：
   ```python
   practice = {
       "title": "测试命名规范",
       "description": "测试方法名应清晰表达测试目的",
       "category": "测试规范"
   }
   practice_id = kb.add_best_practice(practice)
   ```

## 5. 测试用例生成

### 5.1 访问测试用例管理

1. 在左侧导航栏点击"测试用例管理"
2. 进入测试用例管理页面，查看AI生成的测试用例

### 5.2 生成测试用例

1. **通过API生成**：
   ```bash
   curl -X POST http://localhost:8001/api/generate/test-cases \
     -H "Content-Type: application/json" \
     -d '{
       "requirements": "用户登录功能，需要测试正常登录、密码错误、账号不存在等场景",
       "test_type": "E2E",
       "framework": "playwright"
     }'
   ```

2. **通过前端界面生成**：
   - 在测试用例管理页面点击"生成测试用例"按钮
   - 输入需求描述、测试类型和框架
   - 点击"生成"按钮

### 5.3 管理测试用例

1. **预览测试用例**：点击"预览"按钮查看测试用例详情
2. **编辑测试用例**：点击"编辑"按钮修改测试用例内容
3. **评分测试用例**：使用评分组件对测试用例质量进行评分

## 6. 执行监控

### 6.1 访问执行监控

1. 在左侧导航栏点击"执行监控"
2. 进入执行监控页面，包含以下功能：
   - 执行统计信息
   - 实时执行日志
   - 执行时长分布
   - 执行详情表格

### 6.2 查看执行结果

1. **执行统计**：查看成功用例数、失败用例数和总执行时长
2. **实时日志**：查看测试执行的实时日志和错误信息
3. **执行详情**：查看每个测试用例的执行状态、时长和时间戳
4. **失败截图**：点击"查看截图"按钮查看失败测试的截图

### 6.3 分析执行数据

1. **执行时长分布**：查看每个测试用例的执行时长，分析性能瓶颈
2. **成功率分析**：分析测试用例的成功率，找出问题集中的区域
3. **错误类型分析**：分析测试失败的错误类型，找出常见问题

## 7. 缺陷管理

### 7.1 访问缺陷管理

1. 在知识库管理页面查看历史缺陷
2. 使用搜索功能查找特定缺陷
3. 点击"查看"按钮查看缺陷详情

### 7.2 缺陷分析

1. **严重程度分析**：根据严重程度分类缺陷
2. **状态分析**：查看缺陷的解决状态
3. **趋势分析**：分析缺陷的产生和解决趋势

### 7.3 缺陷模式识别

1. **添加缺陷模式**：
   ```python
   defect_pattern = {
       "name": "空指针异常",
       "description": "访问空对象导致的异常",
       "symptoms": "系统崩溃"
   }
   defect_pattern_id = kb.add_defect_pattern(defect_pattern)
   ```

2. **搜索相似缺陷**：
   ```python
   similar_defects = kb.search_similar_defects("登录失败", top_k=5)
   ```

## 8. 最佳实践

### 8.1 测试策略

1. **分层测试**：
   - 单元测试：测试单个组件和函数
   - 集成测试：测试组件之间的交互
   - E2E测试：测试完整的用户流程

2. **测试数据管理**：
   - 使用数据工厂服务生成测试数据
   - 确保测试数据的一致性和可重复性

3. **测试环境管理**：
   - 使用Docker容器化测试环境
   - 确保测试环境与生产环境一致

### 8.2 性能测试

1. **执行性能测试**：
   - 设置性能测试参数（并发用户数、持续时间等）
   - 执行性能测试并收集数据
   - 分析性能测试结果

2. **性能指标监控**：
   - 响应时间
   - 吞吐量
   - 资源使用率（CPU、内存、磁盘I/O等）

### 8.3 安全测试

1. **常见安全测试**：
   - SQL注入测试
   - XSS测试
   - CSRF测试
   - 认证和授权测试

2. **安全测试工具**：
   - OWASP ZAP
   - Burp Suite
   - Nmap

## 9. 故障排除

### 9.1 常见问题

1. **服务启动失败**：
   - 检查端口是否被占用
   - 检查数据库连接是否正常
   - 检查环境变量配置是否正确

2. **测试用例生成失败**：
   - 检查OpenAI API密钥是否有效
   - 检查需求描述是否清晰
   - 检查生成服务是否正常运行

3. **执行测试失败**：
   - 检查测试环境是否准备就绪
   - 检查测试用例是否正确
   - 检查被测系统是否正常运行

### 9.2 日志查看

1. **前端日志**：
   - 浏览器开发者工具控制台
   - 前端应用日志文件

2. **后端日志**：
   - 服务启动终端输出
   - 服务日志文件

3. **数据库日志**：
   - MongoDB日志
   - Milvus日志
